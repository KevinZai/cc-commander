# Content Cluster Strategy

Design topic cluster architectures that dominate search results through topical authority.

## Process

### 1. Identify Pillar Topics
- Analyze target keywords from `seo-context`
- Group into 3-5 broad pillar topics
- Each pillar = a comprehensive guide (2000-5000 words)
- Pillars target high-volume, competitive head terms

### 2. Map Cluster Content
For each pillar, identify 8-15 cluster pages:
- Long-tail keyword variations
- Question-based queries (how, what, why, best, vs)
- Use-case specific content
- Comparison and alternative pages
- Each cluster page targets a specific long-tail keyword
- Each cluster page links back to the pillar

### 3. Design Internal Linking
- Pillar ↔ Cluster (bidirectional)
- Cluster ↔ Cluster (related topics within same pillar)
- Cross-pillar links (sparingly, only when genuinely relevant)
- Generate an internal linking map as a visual reference

### 4. Content Calendar
- Prioritize clusters by: search volume × keyword difficulty × business value
- Recommend publishing cadence
- Phase 1: Pillar pages first (establish authority)
- Phase 2: High-priority clusters
- Phase 3: Long-tail clusters

### 5. Measurement
- Track pillar page rankings (head terms)
- Track cluster page rankings (long-tail)
- Monitor internal link click-through
- Measure topical authority growth over time

## Output
Generate:
1. `content-clusters.md` — Full topic cluster map with keywords, search volumes, and linking structure
2. `content-calendar.md` — Prioritized publishing schedule
3. Internal linking diagram (text-based or mermaid)
