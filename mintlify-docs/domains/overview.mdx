## What Are CCC Domains?

A CCC domain is a **router skill** — one entry point that absorbs many specialist skills and dispatches to the right one based on your intent.

```
/ccc-saas  →  "Add billing" → stripe-subscriptions + billing-automation
           →  "Set up auth" → better-auth
           →  "Design my DB" → database-designer + drizzle-neon
```

You don't need to know which specialist handles your task. Describe your goal. The router figures it out.

## How to Use Any Domain

**Step 1:** Load the domain router.

```
/ccc-saas
```

**Step 2:** Describe your intent. The router identifies the specialist.

```
"Add Stripe subscriptions with annual billing"
→ Routes to: stripe-subscriptions + billing-automation
```

**Step 3:** The specialist handles the work with domain expertise.

No hunting for the right skill. No loading multiple specialists manually. One description, one dispatch.

## Domain Loading via `/init`

When you run `/init`, CCC asks what you're building and auto-selects the right domains:

```
What are you building?
❯ SaaS product          → ccc-saas + ccc-testing + ccc-devops
  Marketing site        → ccc-design + ccc-seo + ccc-marketing
  API / Backend         → ccc-saas + ccc-devops + ccc-security
  Mobile app            → ccc-mobile + ccc-testing
  Data pipeline         → ccc-data + ccc-devops
  Something else        → Browse all 455+ skills
```
