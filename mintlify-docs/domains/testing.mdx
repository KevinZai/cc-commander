## Overview

Load one skill. Get the entire testing domain.

```
/ccc-testing
```

15 skills built from 11 proven testing tools plus 4 new specialists.

## Sub-Skills

| # | Skill | What It Does |
|---|-------|-------------|
| 1 | `e2e-testing` | Playwright E2E — page objects, fixtures, assertions, CI setup |
| 2 | `webapp-testing` | Playwright for local web apps — dev server integration, visual testing |
| 3 | `tdd-workflow` | Test-driven development — red/green/refactor cycle, coverage tracking |
| 4 | `verification-loop` | Comprehensive verification — multi-step validation before marking done |
| 5 | `verification-before-completion` | Proof-of-work verification — evidence that changes work correctly |
| 6 | `ai-regression-testing` | AI-assisted regression — detect regressions using AI analysis |
| 7 | `eval-harness` | Eval-driven development — build evals first, then implement to pass them |
| 8 | `qa` | Full QA pass — find bugs, fix them, commit fixes |
| 9 | `qa-only` | Report-only QA — find and report bugs without fixing them |
| 10 | `plankton-code-quality` | Write-time quality — catch issues as code is written, not after |
| 11 | `python-testing` | Python testing — pytest, fixtures, mocking, TDD for Python projects |
| 12 | `testing-router` | Routes testing tasks to the right specialist |
| 13 | `test-strategy` | Generates a complete testing strategy for any project |
| 14 | `visual-regression` | Screenshot comparison, Percy, Chromatic |
| 15 | `load-testing` | k6/Artillery scripts, performance benchmarks, stress testing |

## Routing Matrix

| Your Intent | Routes To |
|------------|----------|
| "Write tests first / TDD" | `tdd-workflow` |
| "E2E tests / Playwright" | `e2e-testing` + `webapp-testing` |
| "Verify my work is done" | `verification-loop` |
| "Find bugs / QA this" | `qa` (fix) or `qa-only` (report) |

| "Visual regression / screenshot tests" | `visual-regression` |
| "Load testing / performance" | `load-testing` |
| "Build evals" | `eval-harness` |
| "Python tests / pytest" | `python-testing` |

## Full Testing Strategy Flow

```
test-strategy      → analyze project, recommend test types and targets
tdd-workflow       → set up framework, write first tests
e2e-testing        → critical user flow tests
visual-regression  → screenshot tests for UI components
load-testing       → performance benchmarks
```
